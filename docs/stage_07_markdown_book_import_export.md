# 阶段 07：Markdown 图书清单导入与导出

## 目标

在管理员端增加图书清单导入与导出能力，用固定 Markdown 表格格式保存和恢复图书基础信息。

## 完成内容

- 新增 `export_books_to_markdown()`：
  - 仅允许导出 `.md` 文件。
  - 导出当前全部未删除图书。
  - 输出固定 Markdown 表格格式。
- 新增 `import_books_from_markdown()`：
  - 仅允许导入 `.md` 文件。
  - 按固定 Markdown 表格读取图书编号、书名、作者、出版社、出版日期、价格。
  - 导入成功后覆盖当前全部图书信息。
  - 每本导入图书自动生成 3 个可借副本。
  - 信息不全、格式错误、日期错误、价格错误或编号重复的行会被丢弃。
- 管理员界面新增：
  - “导出图书清单”按钮。
  - “导入图书清单”按钮。
  - 导入前覆盖确认提示。

## Markdown 格式

```markdown
| 图书编号 | 书名 | 作者 | 出版社 | 出版日期 | 价格 |
|---|---|---|---|---|---|
| B001 | Python程序设计 | 张三 | 清华大学出版社 | 2021-01-10 | 59.80 |
```

## 导入覆盖规则

- 如果文件中至少存在 1 条有效图书行，则用这些有效行替换当前图书清单。
- 当前 `borrow_records`、`book_copies`、`books` 会被重建。
- 用户账号不会被修改。
- 如果没有任何有效图书行，则不会修改当前数据库。

## 验证重点

- 导出文件应包含固定表头和当前图书信息。
- 导入有效 Markdown 后，图书清单变为导入文件中的有效图书。
- 无效行应被跳过。
- 导入后每本图书都有 3 个可借副本。
- 非 `.md` 文件应被拒绝。
