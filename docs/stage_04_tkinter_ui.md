# 阶段 04：Tkinter 图形界面

## 目标

将已完成的认证和业务服务接入图形化界面，满足普通用户和管理员两类角色的操作流程。

## 完成内容

- `main.py` 启动时自动初始化数据库并打开登录窗口。
- `ui/login_window.py` 实现账号、密码登录和角色跳转。
- `ui/user_window.py` 实现普通用户主界面：
  - 查询图书
  - 借书
  - 还书
  - 我的借阅
  - 退出登录
- `ui/admin_window.py` 实现管理员主界面：
  - 录入图书
  - 删除图书
  - 修改图书信息
  - 查询图书
  - 查询用户借阅状态
  - 总览图书信息
  - 退出登录
- 查询结果使用 `ttk.Treeview` 表格展示。
- 操作成功、失败、无结果等情况使用弹窗提示。

## 界面约定

- 左侧为功能按钮区。
- 右侧为表格结果区。
- 底部为当前状态提示。
- 管理员修改图书时不提供书名输入框，避免修改书名。

## 验证重点

- 登录失败应弹出错误提示。
- 管理员登录后进入管理员界面。
- 普通用户登录后进入普通用户界面。
- 普通用户可借书、还书、查看当前借阅。
- 管理员可录入、删除、修改、查询和总览图书。
